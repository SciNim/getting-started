SciNim Getting Started

Overview of the scientific computing ecosystem

This chapter aims to provide a rough overview of the ecosystem for scientific computing with the aim to present the available packages with an overview of what they can be used for.

In general, keep the Nimble directory handy to search for Nim packages from your browser.

Note that due to the way this page is laid out, some packages might appear multiple times under different sections.

Further: if you feel any existing library is missing on this page, please either create a PR to this page, open an issue or simply write a message in the Matrix/Discord Nim science channel!

Fundamental data handling libraries

The libraries listed here all provide basic data types that are helpful in general (multidimensional arrays & data frames).

  • Arraymancer ‚áê provides a generic Tensor[T] type, similar to a Numpy ndarray. On top it defines operations from indexing, broadcasting and apply/map/fold/reduce operations to linear algebra and much more
  • Neo ‚áê provides primitives for linear algebra. This means it implements vectors and matrices, either with static or dynamic sizes.
  • Datamancer ‚áê builds on top of Arraymancer to provide a runtime based DataFrame implementation. Runtime based means the types of columns are determined at runtime instead of compile time (e.g. via a schema). The focus is on column based operations.
  • NimData ‚áê provides another DataFrame implementation, which - compared to Datamancer - has a stricter CT safety focus. Its implementation is row based and the DataFrame type is determined at compile time. Operations are built on top of iterators for lazy evaluation.

Data visualization

There are multiple libraries for data visualization ("plotting") available, each with their own focus and thus pros and cons.

Beyond the libraries listed in this section, keep in mind that your favorite Python, Julia and R plotting library is only a nimpy, nimjl and Rnim call away!

  • ggplotnim ‚áê pure Nim library for data visualization that is highly inspired by ggplot2 for R. Also see the introduction here.
  • nim-plotly ‚áê an interface to the JavaScript library plotly.js; generates plotly compatible JSON
  • gnuplot.nim ‚áê one of two available bindings to gnuplot
  • gnuplotlib ‚áê one of two available bindings to gnuplot
  • asciigraph ‚áê plots data using unicode symbols to draw pretty graphs in the terminal
  • gr.nim ‚áê wrapper of the GR visualization framework

Numerical algorithms

Numerical algorithms for integration, interpolation, (numerical) differentiation and solving differential equations are of course fundamental for scientific computing.

  • Numericalnim ‚áê the most comprehensive library for numerical algorithms in Nim. Also see the integration tutorial here.
  • Polynumeric ‚áê provides common operations (derivatives, root finding, etc.) of polynomials

Optimization

Optimization (possibly non-linear) problems are a problem domain large enough to deserve their own section beyond the "numerical algorithm" section.

  • Numericalnim ‚áê provides some algorithms for non-linear optimization. These include Levenberg-Marquardt for non-linear curve fitting and (L)BFGS for general optimization problems.
  • fitl ‚áê contains a pure Nim linear least squares solver (so no LAPACK dependency!) and provides many goodness-of-fit tests
  • nimnlopt ‚áê wrapper of the NLopt C library. It includes a large number of algorithms for non-linear optimization problems (gradient / non gradient & local / global methods) with support for constraints.
  • nim-mpfit ‚áê wrapper of the C library cmpfit, an implementation of the Levenberg-Marquardt algorithm for non-linear least squares problems (i.e. non-linear curve fitting).
  • gsl-nim ‚áê wrapper for the GNU Scientific Library, which probably

satisfies all your numerical optimization needs (and much more), if you can live with the GSL dependence and raw C API.

(Binary) data storage & serialization libraries

  • nimhfd5 ‚áê high level bindings for the HDF5 library
  • netcdf ‚áê wrapper for NetCDF library
  • mcpl ‚áê wrapper for MCPL library
  • Arrow
    • freccia ‚áê pure Nim library for Apache Arrow format
    • nimarrow_glib ‚áê wrapper of libarrow
  • nio ‚áê also includes operations for binary data handling
  • nimcfitsio ‚áê wrapper for the CFITSIO library, typically used in astronomy
  • nim-teafiles ‚áê library to read TeaFiles, a format for fast read/write access to time series data
  • CSVtools ‚áê library for typed iterators on CSV files
  • DuckDB ‚áê DuckDB wrapper for Nim. DuckDB is a DB focused on fast data analysis

Linear algebra

A list of libraries for linear algebra operations. These libraries typically provide their own matrix and vector types and define common (and not so common) operations on them.

  • Neo ‚áê linear algebra library with support for dense and sparse matrices. Wraps BLAS & LAPACK and also has GPU support
  • Manu ‚áê pure Nim library for operations on real, dense matrices (solving linear equations, determinants, matrix inverses & decompositions, ...)
  • Arraymancer ‚áê Arraymancer also provides many linear algebra routines
  • gsl-nim ‚áê GSL provides many linear algebra routines

Algebra

  • emmy ‚áê Algebraic structures and operations on them
  • nim-algebra ‚áê implements many routines for rings, fields and groups

Symbolic operations

Libraries dealing with symbolic instead of numeric operations.

  • astgrad ‚áê symbolic derivatives based on Nim AST
  • symbolicnim ‚áê pure Nim library for symbolic computations
  • symengine ‚áê wrapper for C++ library for symbolic computations

Number types

These libraries all provide specific data types suited to certain kind of operations.

  • decimal libraries
    • nim-decimal ‚áê decimal library wrapping C lib mpdecimal
    • decimal128 ‚áê pure Nim decimal library, missing some features
  • multi-precision integers (bigints)
    • bignum ‚áê wrapper of GMP providing arbitrary precision ints & rationals, does not wrap mpfr (so no multi precision floats)
    • bigints ‚áê pure Nim bigint library
    • theo ‚áê optimized bigint library, WIP
  • fpn ‚áê fixed point number library in pure Nim
  • stdlib rationals ‚áê Nim standard library module for rational numbers
  • stdlib complex ‚áê Nim standard library module for complex numbers

Statistics, sampling and random number generation

  • statistical-tests
  • linear-models
  • distributions
  • stdlib stats ‚áê basic statistics module from the stdlib. Supports moments up to kurtosis & provides basic regression support
  • alea ‚áê library for sampling from many different distribiutons. Allows to wrap custom (e.g. stdlib) RNGs
  • sitmo ‚áê Nim implementation of the Sitmo parallel RNG
  • stdlib random ‚áê random number generation of the Nim standard library
  • nim-random ‚áê alternative to the Nim stdlib random number library
  • nim-mentat ‚áê implements exponentially weighted moving averages
  • fitl ‚áê contains a submodule dists for random sampling, mixing of distributions, (inverse) CDFs

Machine learning

  • Flambeau ‚áê as a wrapper to libtorch provides access to state-of-the-art ML features
  • Arraymancer ‚áê Arraymancer implements a DSL to define neural networks (see the examples) and provides other, more primitive ML tools (PCA, ...)
  • exprgrad ‚áê Experimental deep learning framework, based on an easily extensible LLVM compiled differentiable programming language
  • DecisionTreeNim ‚áê implements decision trees & random forests

Natural language processing

  • tome ‚áê provides tokenization and parts of speech (POS) tagging
  • word2vec ‚áê Word2vec implementation in Nim
  • fastText ‚áê library to perform predictions of fastText models
  • scim ‚áê library for helpful tools for speech recognition based on arraymancer

Spatial data structures, distance measures & clustering algorithms

  • kdtree ‚áê k-d tree implementation in pure Nim
  • RTree ‚áê R- and R*-Tree implementations in pure Nim
  • QuadtreeNim ‚áê Quadtree implementation implementation in pure Nim
  • distances ‚áê library to compute distances under different metrics with support for standard sequences, arraymancer & neo types
  • arraymancer ‚áê arraymancer contains a k-d tree implementation, multiple distance metrics (incl. user defined custom metrics) plus k-means & DBSCAN clustering algorithms
  • spacy ‚áê collection of different spatial data structures
  • DelaunayNim ‚áê library to compute the Delaunay triangulation of a set of points
  • nim-mentat ‚áê implements Balanced Box-Decomposition trees

Special functions

These libraries implement different special functions.

  • stdlib math ‚áê The Nim standard library math module contains all libraries you find in math.h.
  • spfun ‚áê library for many special functions used in stats, physics, ...
  • gsl-nim ‚áê The GSL probably provides any special function you may need
  • special-functions ‚áê contains many special functions, which are not part of the stdlib module

FFT

  • nimfftw3 ‚áê FFTW3 wrapper
  • impulse ‚áê pocket FFT wrapper, in principle a repository for signal processing primitives
  • kissFFT ‚áê kissFFT wrapper

Primitive compute wrappers

  • nimcuda ‚áê wrapper for CUDA
  • nimblas ‚áê wrapper for BLAS
  • nimlapack ‚áê wrapper for LAPACK
  • nimcl ‚áê wrapper for OpenCL

Multithreading, multiprocessing & asynchronous processing

  • weave ‚áê very low overhead, high performance multithreading runtime
  • taskpools ‚áê lightweight threadpool implementation
  • threadpools ‚áê Custom threadpool implementation
  • threading ‚áê New pieces for multithreading in times of ARC/ORC
  • asynctools ‚áê Various async tools for usage with Nim's stdlib async macro
  • asyncthreadpool ‚áê An awaitable threadpool implementation
  • cligen ‚áê contains a procpool submodule for easy multiprocessing

Biology

  • hts-nim ‚áê A wrapper for htslib for Nim for parsing of genomics data files
  • bionim ‚áê collection of data structures and algorithms for bioinformatics
  • bio ‚áê a library for working with biological sequences

Physics & astronomy

  • unchained ‚áê library for CT checking of physical units and automatic conversion between units
  • qex ‚áê lattice QCD library
  • mclimit ‚áê Nim port of the ROOT TLimit class for confidence level computations (limits) for experiments with small statistics
  • nim-constants ‚áê contains many physical and mathematical constants
  • astroNimy ‚áê astronomical image processing library
  • orbits ‚áê library for orbital mechanics calculations
  • nim-root ‚áê partial wrapper for CERN's ROOT
  • MDevolve ‚áê integrator framework for molecular dynamic evolutions
  • polypbren ‚áê program to compute renormalized parameters of charged colloids
  • xrayAttenuation ‚áê for calculations of X-ray transmissions through materials & X-ray reflectivity under grazing angles

Mathematics

  • perms-nim ‚áê library for permutation group calculations and factorization algorithms

Other useful libraries

  • scinim ‚áê library of general scientific things that are either primitives or too small to have their own library
  • Measuremancer ‚áê library for automatic error propagation of measurement uncertainties
  • gsl-nim ‚áê wrapper for GSL (GNU Scientific Library)
  • nim-opencv ‚áê Nim wrapper for OpenCV
  • zero-functional ‚áê library for zero cost chaining of functional primitves (map, apply, fold, ...). Fuses multiple operations into a single loop.
  • iterrr ‚áê another library for zero cost chaining, similar to zero-functional. Aims to be easier to extend.
  • flower ‚áê pure Nim bloom filter, probabilistic data structure to check if elements are in a set ("possibly in set" vs. "definitely not in set"). Supports arbitrary Nim types in single filter.
  • pattern matching:
    • pattern matching in fusion ‚áê pattern matching for Nim. Possibly the most feature rich pattern matching library for Nim. Future developments might be found here
    • patty
    • gara
  • Synthesis ‚áê DSL to generate statically checked state machines
  • jupyternim ‚áê Jupyter kernel for Nim
  • cligen ‚áê elegant library to write CLI interfaces
  • LatexDSL ‚áê DSL to generate CT checked latex strings, supporting Nim variable interpolation
  • nim-mathexpr ‚áê mathematical string expression evaluator library
  • nim-pari ‚áê wrapper for the PARI C library underlying the PARI/GP computer algebra system
  • memo ‚áê macro library to allow memoization of function calls (automatic caching of function calls)
  • forematics ‚áê Nim implementation of a Metamath verifier
  • DeepLearningNim ‚áê example of building a DQN with arraymancer

Educational resources

  • nim-bayes ‚áê Tutorial about Bayesien Inference of a linear model in Nim

Language bindings

First of all Nim itself of course provides direct support to wrap C and C++ libraries using its FFI. See the Nim manual here for an introduction to the C / C++ FFI.

For more details on how to use the language specific bindings, see the section about it here

Direct language bridges

  • nimjl ‚áê bridge to Julia
  • nimpy ‚áê bridge to Python
  • Rnim ‚áê bridge to R

Tools to wrap C / C++

  • c2nim ‚áê the default Nim tool to generate Nim wrappers of C header files
  • futhark ‚áê automatic imports of C header files in Nim code
  • nimterop ‚áê library to simplify wrapping of C/C++ using tree-sitter
